Language

Models in gEcon are written using language suited for formulating economic dynamic optimisation problems. gEcon model file consists of blocks describing agents optimisation problems, exogenous shocks and market clearing conditions. Within each optimisation block the user specifies objective, control variables and constraints of particular agent. Blocks can also be created to impose market clearing conditions and describe the behaviour of exogenous variables.

Parameters of models can be set either in a .gcn file or later on using R interface. Parameters can be also calibrated based on user supplied steady state relationships between variables.

R class gecon_model

gEcon has user friendly interface written in R. Model file created by the shared library is loaded into R environment and an object of gecon_model class is created. Model can be solved and analysed using such gEcon features as:

  • automatically created LaTeX documentation,
  • calibration utilities,
  • steady state and perturbation solvers,
  • tools for IRFs and statistics computations.
gEcon interface is interactive, enabling users to change settings, parameters "on the fly" without the need to recompute the whole model. Moreover gEcon is equipped with a set of tools for analysing model specification, which can be useful in debugging or searching for the steady state.

Numerical solvers employed

  • nleqslv for solving systems of nonlinear equations using Newton or Broyden methods with line searches,
  • gensys for solving perturbations of the first order.
