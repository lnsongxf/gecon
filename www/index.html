About gEcon

gEcon is a framework for developing and solving large scale dynamic (stochastic) & static general equilibrium models. It consists of model description language and an interface with a set of solvers in R. It was developed at the Department for Strategic Analyses at the Chancellery of the Prime Minister of the Republic of Poland as a part of a project aiming at construction of large scale DSGE & CGE models of the Polish economy.

Publicly available toolboxes used in RBC/DSGE modelling require users to derive the first order conditions (FOCs) and linearisation equations by pen & paper (e.g. Uhlig’s tool-kit) or at least require manual derivation of the FOCs (e.g. Dynare). Derivation of FOCs is also required by GAMS and GEMPACK — probably the two most popular frameworks used in CGE modelling. Owing to the development of an algorithm for automatic derivation of first order conditions and implementation of a comprehensive symbolic library, gEcon allows users to describe their models in terms of optimisation problems of agents. To authors' best knowledge there is no other publicly available framework for writing and solving DSGE & CGE models in this natural way. Writing models in terms of optimisation problems instead of the FOCs is far more natural to an economist, takes off the burden of tedious differentiation, and reduces the risk of making a mistake. gEcon allows users to focus on economic aspects of the model and makes it possible to design large-scale (100+ variables) models. To this end, gEcon provides template mechanism (similar to those found in CGE modelling packages), which allows to declare similar agents (differentiated by parameters only) in a single block. Additionally, gEcon can automatically produce a draft of LaTeX documentation for a model.

The model description language is simple and intuitive. Given optimisation problems, constraints and identities, computer derives the FOCs, steady state equations, and linearisation matrices automatically. Numerical solvers can be then employed to determine the steady state and approximate equilibrium laws of motion around it.

About current release

Version 0.9.0 (NEW!)

gEcon 0.9.0 was released on February 28, 2015. This version introduces the so-called references allowing users to formulate problems constrained by solution to other agents’ problems (eg. Nash bargaining, optimal policy).

Important note to the users of versions prior to 0.8.0: Changes have been made in the names of the functions and classes in the R interface that relate to model parameters. The goal of those changes was to assure consistency with the naming convention for functions and classes pertaining to variables. The following functions and classes have changed their names:
get_free_parameter_names to get_par_names (with extended functionality),
get_parameter_vals to get_par_values,
parameter_info to par_info,
gecon_parameter_info to gecon_par_info.
In addition, names of the following four functions have been changed:
find_eq to list_eq,
find_calibr_eq to list_calibr_eq,
set_shocks to set_shock_cov_mat,
compute_corr to compute_moments.
Please adjust your R code accordingly.

Important note to the users of versions prior to 0.7.0: models written for versions prior to 0.7.0 will not work in the same way or may even fail. Changes include:

  • shocks listed in the .gcn file should now be separated by commas (,) instead of semicolons (;),
  • gEcon now inserts Lagrange multipliers for constraints automatically — explicitly declared Lagrange multipliers will not be automatically selected for reduction, and declarations of Lagrange multipliers on time aggregators in static problems will cause an error.
In order to force backward compatibility use backwardcomp option in the options section of your .gcn model file.
R interface has also been slightly changed. In particular shock list provided as an argument to functions: random_path, compute_irf, simulate_model and gecon_simulation class constructor should now be called shock_list instead of eps_list. Please adjust your R code accordingly.

Plans for development

Current work is focused on the block references, which will make it easy to solve models with Stackelberg competition, Nash bargaining, and Ramsey (benevolent government) problems.

An add-on package allowing for Bayesian estimation of models will be released by the end of February.

An add-on package simplifying calibration of CGE models using SAM matrices will also be released by the end of February.
