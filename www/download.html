gEcon package

Please note that gEcon is NOT DISTRIBUTED THROUGH CRAN. You have to download the .zip (Windows), .tgz (Mac), or .tag.gz (source, Linux / Un*x) file, save it on your disk, and install gEcon package from this file.

Download version 0.8.3 from R-forge

  • gEcon_0.8.3.tar.gz [1 MB] — source code (Linux / Un*x systems)
  • gEcon_0.8.3.zip [2.5 MB] — binary version for Windows
  • gEcon_0.8.3.tgz [1.4 MB] — binary version for Mac (built under OS X 10.9.5)

Important note to the users of versions prior to 0.8.0: Changes have been made in the names of the functions and classes in the R interface that relate to model parameters. The goal of those changes was to assure consistency with the naming convention for functions and classes pertaining to variables. The following functions and classes have changed their names:
get_free_parameter_names to get_par_names (with extended functionality),
get_parameter_vals to get_par_values,
parameter_info to par_info,
gecon_parameter_info to gecon_par_info.
In addition, names of the following four functions have been changed:
find_eq to list_eq,
find_calibr_eq to list_calibr_eq,
set_shocks to set_shock_cov_mat,
compute_corr to compute_moments.
Please adjust your R code accordingly.

Important note to the users of versions prior to 0.7.0: models written for versions prior to 0.7.0 will not work in the same way or may even fail. Changes include:

  • shocks listed in the .gcn file should now be separated by commas (,) instead of semicolons (;),
  • gEcon now inserts Lagrange multipliers for constraints automatically — explicitly declared Lagrange multipliers will not be automatically selected for reduction, and declarations of Lagrange multipliers on time aggregators in static problems will cause an error.
In order to force backward compatibility use backwardcomp option in the options section of your .gcn model file.
R interface has also been slightly changed. In particular shock list provided as an argument to functions: random_path, compute_irf, simulate_model and gecon_simulation class constructor should now be called shock_list instead of eps_list. Please adjust your R code accordingly.
